Which programming language to choose?

The simulator currently offers two programming languages: rEduc and C#. It is possible to program rEduc in a text-based environment such as visual studio code as well as in blockly (using our software called BlockEduc).

rEduc is simple and intuitive, and having the possibility to program in blocks as well as text makes it the preferable option to most users familiar with LEGO kits. C# offers more options such as Threads and access to more advanced functionalities and variables of the robot, but is slightly more complicated and its syntax harder than rEduc.
We personally recommend rEduc for programming: it is simple, intuitive, easy to read and will be sufficient for most of the user's needs. However, if you already have experience in another programming language, specially C++, Java or C, C# can also be an option for you as rEduc may be "limiting" for your needs.

Another advantage of programming using text is the support for external editors through our import/export code feature.
There are a couple of snippets for rEduc/C# however our community is developing an API-based extension for VSCode and Sublime Text that will update with the simulator.

Code

arenas:code

The arena code is one of the core features of the platform. With it one can generate the same arena as another and thus be able to import or export fields with ease. These codes usually seem complex and hard to understand, and most of its features can be found on the Arena Editor. However, some users may find the code easy to manipulate and understand and may prefer to create their own arena-manipulating platforms.

Settings

General settings

(String) time_of_day

A string from 6:00 to 18:00, where the second number can vary from 00 to 59

Example: "time_of_day": "14:00",
(String) description

Description that any user who imports the arena can see on the top-right corner of the "testing area" of the simulator.

Example: "description": "Untitled Arena",
(Boolean) robot_has_to_follow_line

The most important boolean in the arena settings. This generates the line the robot has to follow and with this the user can activate Rescue Kits, Markers, Standard RoboCupJr scoring, and many other settings.

Example: "robot_has_to_follow_line": true,
(Boolean) time_limit_stops_code_execution

Stops the execution of the user's code after the time limit (see time_limit below) is exceeded.

Example: "time_limit_stops_code_execution": true,
(Boolean) finish_execution_after_code

Stops the simulation after the last line of code (instead of standing still like in the normal behavior).

Example: "finish_execution_after_code": false,
(Boolean) change_position_in_routine

Allows the user to change the robot's position manually while the simulation is running.

Example: "change_position_in_routine": false,
(Integer) obstacle_dodging_time

Integer from 10 to 120 (seconds) of how long the robot has to dodge an obstacle before a lack of progress is called.

Example: "obstacle_dodging_time": 50,
(Boolean) obstacle_pushing

Sets whether or not a lack of progress will be called when the robot pushes an obstacle.

Example: "obstacle_pushing": true,
(Boolean) distance_based_checkpoint_scoring

Sets whether or not the marker scoring algorithm is going to consider how far a marker is from another.

Example: "distance_based_checkpoint_scoring": true,
(Boolean) standard_robocup_jr_scoring

Sets whether or not the arena will use multipliers and other international scoring features instead of their Brazilian counterparts.

Example: "standard_robocup_jr_scoring": true,
(Boolean) rescue_kit_enabled

Enables the rescue kit. Depends on level_two_rescue_kit to set the level.

Example: "rescue_kit_enabled": true,
(Boolean) lack_of_progress

Enables or disables completely the lack of progress feature.

Example: "lack_of_progress": true,
(Integer) number_of_checkpoints

Sets how many user-input checkpoints are going to be present in the arena (Pre-2021 rules/Brazilian ruleset).

Example: "number_of_checkpoints": 0,
(Integer/Enum) time_limit

This may change in the future, however currently the simulator uses the enumerator below for the time limits:

# TimeLimits
0 OneMinute
1 TwoMinutes
2 ThreeMinutes
3 FiveMinutes
4 EightMinutes
5 TenMinutes
6 NoTimeLimit

Example: "time_limit": 4,
(Boolean) level_two_evacuation_point

Sets the Evacuation Point's level (true equals "level 2").

Example: "level_two_evacuation_point": false,
(Boolean) level_two_rescue_kit

Sets the Rescue Kit's level (true equals "level 2").

Example: "level_two_rescue_kit": false,

Rescue Zone settings

(Integer) live_victims

Sets the number of live victims inside the Rescue Zone (0 to 10).

Example: "live_victims": 2,
(Integer) dead_victims

Sets the number of dead victims inside the Rescue Zone (0 to 10).

Example: "dead_victims": 1,
(Integer/Enum) ramp

Ramp is an integer that references an Enum. This defines the uniquely Brazilian Rescue Zone ramp.

# Ramps
0 NoRamp
1 Line
2 Gap
3 SpeedBump
4 MultipleSpeedBumps
5 MultipleGaps
6 GapAndSpeedBump

Example: "ramp": 0,
(Integer/Enum) exit

Exit is an integer that references an Enum. This defines in which position the Rescue Zone exit will be generated.

# Exits
0 NoExit
1 LeftFacing
2 RightFacingFrontal
3 RightFacing

Example: "exit": 1,
(Integer/Enum) point_position

Point_position is an integer that references an Enum. This defines in which position the Evacuation Point exit will be placed.

# EPPos
0 NoEvacuationPoint
1 Random
2 LeftFacing
3 RightFacingFrontal
4 RightFacing

Example: "point_position": 1,

Permissions

(Boolean) robot_one

Enables or disables the permission to use said robot.

Example: "robot_one": true,
(Boolean) robot_two

Enables or disables the permission to use said robot.

Example: "robot_two": true,
(Boolean) robot_three

Enables or disables the permission to use said robot.

Example: "robot_three": true,
(Boolean) robot_four

Enables or disables the permission to use said robot.

Example: "robot_four": true,
(Boolean) robot_five

Enables or disables the permission to use said robot.

Example: "robot_five": true,
(Boolean) custom_robots

Enables or disables the permission to use said robot.

Example: "custom_robots": true,

Tiles

Each tile property and field is further discussed here.

Example of a tile:
{
 "id": 6,
 "x": 6,
 "z": 3,
 "y": 1,
 "type": "curve",
 "rotation": 0,
 "properties": {
  "has_checkpoint": false
 }
}

General fields

(Integer) id

Sets the id of the tile.

Example: "id": 0,
(Integer) x

Sets the x position of the tile.

Example: "x": 1,
(Integer) z

Sets the z position of the tile.

Example: "z": 1,
(Integer) y

Sets the height of the tile.

Example: "y": 0,
(String) type

Sets the type of the tile. Types can either be:

  • straight_line
  • curve

Example: "type": "straight_line",
(Integer) rotation

Sets the rotation of the tile.

# Rotation

(Boolean) has_checkpoint

Activates marker (checkpoint) on said tile. However, that is appliable only for non-tileset non-scoring tiles. The property will be ignored if activated on an invalid tile.
